Pano Image Build and Debug

Skip to end of metadata
  • Created by Yang, Ying, last modified by Chen, Shawn on Nov 07, 2017
Go to start of metadata

This page introduces the steps to build Pano Image and debug Pano with Android Studio.


  1. Set up Developer VM(Details in Developer Workstation Virtual Machines)  
    1.1 SSH
          Your VM Domain is xxx-dev.polycom.com, (xxx is your ID, the short name of your account, not your email), your can login in with any SSH client (Putty, etc.)
          e.g. 
          Commond:
            ssh yiyang@yiyang-dev.polycom.com
          or SSH Client: Putty
            

    1.2 NoMachine    
          1.2.1 Download the client for Windows, Mac, or Linux.
          1.2.2 Connect your dev-vm.
                  
         1.2.3 Configuration
                 Note: To resize the desktop contents of the NoMachine GUI window, move your mouse to the top right corner until the menu icon appears for an paper corner upturned.
      1.3  Manager your Dev-vm
             Visit https://vmware.polycom.com/   >  vSphere Client (HTML5) - partial functionality   (sign in with your ID: <short name>@polycom.com)
             
             After that, your can search your vm with your ID:
             
         Note: if you can't sign in with NoMachine using correct username/password, please restart your VM and try again.
         
        
         when you sign in ubuntu, you can see the UI like this:
        
  2. Build your Pano Image(Details in Rockstars Build Instructions)
    2.1 Download source code with svn
          Open Terminal,
    cd /home/yiyang/
          mkdir source obj
          cd source
          svn co https://aussubversion.polycom.com/svn/rockstars/main/branches/pctc-yiyang
          
    2.2 Build
          cd  ../obj     (Empty Dir)
          ../source/pctc-yiyang/configure --enable-bld-avc
          or with version
          ../source/pctc-yiyang/configure --enable-bld-avc --with-bld-ver=1.1.2
        build options explain:
        There are several build options, like --enable-bld-avc, --enable-bld-app-svcs, --enable-bld-tvui etc, these options are not mutual exclusive, you can 
        select multipule of them according to which module you are working. If you want to build a full version of workable image, the suggested option combination is:
        --enable-bld-avc  --enable-bld-app-svcs  --enable-bld-tvui --enable-bld-webui

  3.       then
          ./build production
          when build success, you can find polycom-pano-***.zip
          
    2.3 Copy File to Local(Details in Updating Software on Hendrix)
          FileZilla is a good tool. You can copy file through sftp.

      2.4 Update Pano
          When you get your package file (polycom-pano-***.zip), copy to your USB-stick in ROOT DIR.
          Plug into Pano USB port, the Pano will find your zip file automatically.
          
          
       If you don't have an USB, you can also update your system by webui. Our dev VM has already started a web service by default.
       you can create a www directory in your home dir, and link your build object folder to a sub dir of www folder, like:
       ln -s ~/work/richards/obj/  ~/www/richards
       Then you can visit this folder by a link like : http://scchen-dev.polycom.com/~scchen/richards/
       You can update it by WebUI's upgrade page.

          If you get a NEW Pano, maybe you need convert from production mode to dev mode，the update zip url is 
          http://builds.austin.polycom.com/vsgbuilds/Rockstars/Development/Hendrix/TestBuilds/1000159/  ,
          and then update your dev version.
          If your Pano is in dev mode, you can update directly from your build zip.
          
          You can change the value of BLD_VER  by editing the file configure.ac, Details in Rockstars configure options
          
    1. Set up Android Studio (Details in Android Studio on Linux for Pano)
      3.1 Download
            Visit: https://developer.android.com/studio/index.html , download windows/ubuntu version:
            in Android Studio 2.3.3
            
            in Android Studio 3.0
            
            in Linux:
            cd /home/yiyang/
            mkdir -p work/android/ide/  work/android/sdk/
            cd work/android/ide/
            wget https://dl.google.com/dl/android/studio/ide-zips/2.3.3.0/android-studio-ide-162.4069837-linux.zip
            unzip android-studio-ide-162.4069837-linux.zip
            ln -s android-studio-ide-162.4069837-linux/android-studio  android-studio
            in windows:
            Download link: https://dl.google.com/dl/android/studio/install/2.3.3.0/android-studio-bundle-162.4069837-windows.exe
            Next,next...or custom your setup path and SDK path. etc.
            Android Studio 3.0,Download https://dl.google.com/dl/android/studio/install/3.0.0.18/android-studio-ide-171.4408382-windows.exe

      3.2 Configuration and run
            in Linux:
            vi ~/.bashrc
            add following text to .bashrc bottom. 

               export ANDROID_IDE=~/work/android/ide/android-studio
               export ANDROID_SDK=~/work/android/sdk
               export PATH=$PATH:$ANDROID_SDK/platform-tools/:$ANDROID_SDK/tools:$ANDROID_IDE/bin:$ANDROID_IDE/gradle/gradle-3.2/bin

            source ~/.bashrc

            studio.sh & 
            Other details, please see:(Android Studio on Linux for Pano)
            in Windows:
            SDK manager:
             
            SDK Platform MUST BE Android 6.0
            
            Need NDK and LLDB and optionally install Python Plugin in setting progress.
            if you download newest trunk code (after file "build.gradle" rev 7157, please update plugins: Android SDK Build-tools 26.0.2, LLDB 3.0 ,Gradle 3.0)
            
             
            
            
            
            Env PATH setup(Please according to your actual path)
            e.g.
           Android SDK Env in Windows 10:
             
            Path Env in Windows 10:
             
            

  4. Debug with IDE
    adb connect 10.250.54.147  (your pano ip)


    adb devices

    add a breakpoint in  ui\android\apps\Home\src\main\java\polycom\home\HomeActivity.java   Line:272

    debug: Menu→Run → Debug → select "Home"

    Waiting...and you will go to your breakpoint
